8.6 CSS3 选择器：找人的艺术 🕵️‍♂️

以前我们找元素，只能靠 id 或 class。这就像在全校广播：“三年二班的王小明同学，请到教务处！”

但是如果元素没有 class 怎么办？

CSS3 选择器 就像给了你一个超级雷达，你可以这样找人：

  • 找“张老师身后的第三个学生”。
  • 找“所有穿红色外套且带有指定 Logo 的学生”。

更精准的定位，意味着你可以少写很多无聊的 HTML class 命名！

8.6.1 伪类：元素的“情绪状态” 🎭

什么叫伪类 (Pseudo-classes)？它通常用单冒号 : 表示。

你可以把它理解为一个元素的特定状态或特定位置。

  • :hover —— 鼠标悬停状态（元素感到“痒”了）。
  • :focus —— 输入框获得焦点的状态（聚光灯打在它身上）。
  • :nth-child(n) —— 找到队伍里特定位置的元素。

👨‍💻 任务 1：给表格换上斑马线衣服

为了让长长的表格更好看，我们希望奇数行和偶数行颜色不一样。

任务代码：使用 :nth-child

/* 选中所有偶数行 (even) 改变背景色 */
tr:nth-child(even) {
    background-color: #f2f2f2;
}

/* 选中所有奇数行 (odd) */
tr:nth-child(odd) {
    background-color: #ffffff;
}

预期效果：无需给每个 tr 写 class，表格自动变成灰白相间的斑马纹！

8.6.2 上下文选择器：理清亲戚关系 👨‍👦

网页里的标签也是有层级和辈分的，找对辈分才能精准施法。

  • 后代选择器 (空格)：A B —— 查找 A 里面所有的 B。（儿子、孙子、曾孙子全算！）
  • 子代选择器 (>)：A > B —— 查找 A 的亲生儿子 B。（孙子不算哦！）
  • 相邻兄弟选择器 (+)：A + B —— 查找紧挨在 A 后面的亲兄弟 B。

👨‍💻 任务 2：只选中亲儿子

假设有一个多级菜单，我们只想让第一层菜单项变红。

<ul class="menu">
    <li>一级菜单
        <ul> <li>二级菜单 (不要变红)</li> </ul>
    </li>
</ul>

/* ❌ 错误做法：所有 li 都变红了 */
.menu li { color: red; }

/* ✅ 正确做法：只找亲儿子 (子代选择器) */
.menu > li { color: red; }

8.6.3 属性选择器：看标签认人 🏷️

有时候我们不想加 class，但发现元素自带一些特殊的属性。比如 <input type="password">。

语法： [属性名="属性值"]

  • [disabled]：选中所有被禁用的按钮。
  • input[type="text"]：只选中普通的文本输入框。
  • a[href^="https"]：选中所有链接开头是 https 的超链接（安全链接）。

👨‍💻 任务 3：让密码框亮起红灯

表单里有很多输入框，我们只想给密码框加个红色的边框警告用户。

<input type="text" placeholder="用户名">
<input type="password" placeholder="密码">

/* CSS 代码：直接利用 type 属性狙击密码框 */
input[type="password"] {
    border: 2px solid red;
    background-color: #ffe6e6;
}

效果：只有密码框的背景和边框变了！没写一个 class！

8.6.4 伪元素：无中生有之术 👻

伪元素 (Pseudo-elements) 通常用双冒号 :: 表示。

它可以直接用 CSS 在页面上凭空捏造出内容，不需要修改 HTML 文件！

  • ::before —— 在元素内容的最前面插入一个“幽灵”。
  • ::after —— 在元素内容的最后面插入一个“幽灵”。

极其重要的咒语：召唤伪元素必须使用 content 属性，哪怕它是空的 content: ""，否则幽灵不会显形！

👨‍💻 任务 4：用伪元素自动加引号

我们要在名言警句的前后自动加上漂亮的巨型引号。

<p class="quote">前端开发是世界上最酷的工作</p>

/* CSS 召唤伪元素 */
.quote::before {
    content: "“";
    color: #1a73e8;
    font-size: 40px;
}
.quote::after {
    content: "”";
    color: #1a73e8;
    font-size: 40px;
}

8.7 CSS3 过渡 (transition) 🏃‍♂️

没有过渡的网页是什么感觉？就像你前一秒在卧室，下一秒瞬间瞬移到了厨房。太突兀了！

transition 就是给状态的改变加上平滑的动画效果。

核心四大要素：

  1. 要过渡的属性（比如 background-color 或 all 代表全部）。
  2. 持续时间（比如 0.3s 秒）。
  3. 运动曲线（比如平滑的 ease 或匀速的 linear）。
  4. 延迟时间（比如 0s，通常省略）。

👨‍💻 任务 5：制作会呼吸的按钮

给按钮加上悬停效果，要求颜色和大小变化是缓缓发生的。

.btn {
    background-color: #1a73e8;
    color: white;
    padding: 10px 20px;
    /* 关键魔法：所有属性变化，耗时0.3秒，平滑 */
    transition: all 0.3s ease; 
}

.btn:hover {
    background-color: #d81b60; /* 变色 */
    transform: scale(1.1);     /* 放大 1.1倍 */
}

⚠️ 老师的叮嘱：transition 必须写在元素本来的 class 里，千万别写在 :hover 里，否则鼠标移开时就没有动画了！

8.8 CSS3 动画 (animation) 🎬

过渡 (transition) 只能实现 A 到 B 的单次变化，且必须要有触发条件（如 hover）。

如果你想让元素一打开网页就不停地自己跳舞，你需要请出动画导演：animation。

拍一部动画片分两步：

  1. 写剧本：使用 @keyframes 定义动画的关键帧。
  2. 演员就位：给元素添加 animation 属性绑定剧本。

8.8 剧本该怎么写？ 📜

使用百分比来控制时间轴，告诉演员在什么进度做什么动作。

/* 剧本名：dance */
@keyframes dance {
    0%   { transform: rotate(0deg); }   /* 开场 */
    50%  { transform: rotate(180deg); } /* 演到一半 */
    100% { transform: rotate(360deg); } /* 结局 */
}

常见的动画属性简写格式：
animation: 剧本名称 时长 曲线 延迟 次数 方向;

例如：animation: dance 2s linear infinite; (无限循环)

👨‍💻 任务 6：让小球永动弹跳

编写一个剧本，让一个红色的圆形小球上下反复弹跳。

/* 1. 写剧本：命名为 jump */
@keyframes jump {
    0%   { transform: translateY(0); }
    50%  { transform: translateY(-50px); }
    100% { transform: translateY(0); }
}

/* 2. 开拍：应用给小球 */
.ball {
    width: 50px; height: 50px; border-radius: 50%;
    background: red;
    /* 绑定jump剧本, 时长1s, 匀速, 无限循环 */
    animation: jump 1s linear infinite;
}

8.9 媒体查询 (Media Queries) 📱💻

你的网页在电脑屏幕上看着很美，但在手机上却挤成了一锅粥？

我们需要响应式设计，而它的灵魂就是 media queries。

媒体查询就像一个变色龙：如果屏幕变窄了，我就换一套 CSS 衣服穿。

  • max-width：如果屏幕宽度小于等于这个值，执行代码。（桌面端优先设计法）。
  • min-width：如果屏幕宽度大于等于这个值，执行代码。（移动端优先设计法，现代大厂首选！）。

👨‍💻 任务 7：电脑左右并排，手机上下堆叠

实现最经典的响应式布局。

/* 默认：电脑端样式 (左右各占一半) */
.box { 
    width: 50%; 
    float: left; 
}

/* 魔法条件：当屏幕宽度小于等于 768px 时 */
@media screen and (max-width: 768px) {
    .box {
        width: 100%; /* 占满整行，变成上下堆叠 */
        float: none;
    }
}

效果：拖动浏览器边缘缩小窗口，布局会瞬间改变！

8.10.1 常用长度单位大作战 📏

我们在写 width 或 font-size 时，到底该用什么单位？

  • px (像素)：绝对单位。你写 20px，在任何设备上都是绝对的 20px。缺点是不够灵活。
  • % (百分比)：相对单位。相对于父元素的尺寸。常用于布局宽度（如 width: 50%）。
  • em：相对单位。相对于当前元素或父元素的字体大小。缺点是嵌套深了容易发生“连环相乘”的惨案（比如子代越来越大）。

8.10.2 新王者：rem (root em) 👑

为了解决 em 连环相乘的混乱，CSS 引入了 rem。

rem 中的 r 代表 root（根节点，即 html 标签）。

原理：

不管你把 rem 写在网页的哪个角落，它都只听命于最高指挥官（html）的 font-size。最高指挥官说是 16px，那么 1rem 就是 16px。

好处：牵一发而动全身！配合媒体查询修改 html 的字体大小，全站的元素都能等比例放大或缩小！

👨‍💻 任务 8：用 rem 实现全局缩放

定好根节点基准，其他地方用 rem 做乘法。

/* 1. 设置根元素基准：1 rem = 16px */
html { font-size: 16px; }

/* 2. 当屏幕大于 1200px 时，放大基准 */
@media (min-width: 1200px) {
    html { font-size: 20px; } 
}

/* 3. 业务代码中使用 rem */
h1 { font-size: 2rem; } /* 手机端是32px，大屏自动变40px！ */
p  { font-size: 1.5rem; }

8.11 图像解决方案：应用背景 🖼️

现代网页中充斥着大量的图标（点赞心形、微信Logo、返回箭头）。

如果每个图标都用独立的 <img src="...png">：

  • 性能噩梦：浏览器要向服务器发起几十上百次 HTTP 请求，页面加载极慢。
  • 维护困难：如果老板说“把所有灰色图标换成蓝色”，美工得重新切几十张图。

前端老司机们为了解决这个问题，发明了一系列神器方案。

8.11.2 CSS Sprites (雪碧图/精灵图) 🥤

生活比喻：去超市买 10 件小商品，你是跑 10 趟超市，还是一次拿个大袋子全装回来？

技术原理：

把所有小图标拼成一张大图（大袋子），网页只需要请求这 1 张图片。

使用时，利用 CSS 的 background-image 和 background-position，就像透过一个小窗户一样，只暴露出你需要的那一个小图标部分。

现状：老旧项目中常见，现代项目已较少使用（放大容易模糊）。

8.11.3 字体图标 (Icon Fonts) 🅰️

既然图片放大不仅模糊且改颜色麻烦，聪明的程序员想：能不能把图标做成文字（字体）？

因为文字有两大绝对优势：

  1. 它是矢量的，怎么放大都不会出现马赛克模糊。
  2. 它属于文本，可以直接用 CSS 的 color 属性任意改变颜色！

目前国内最强大的字体图标库是：阿里巴巴矢量图标库 (Iconfont)。

8.11.4 终极武器：SVG 矢量图 ⚔️

字体图标很好，但它通常只能是单色的（彩色支持较弱）。如果你想要炫酷的多色图标呢？

SVG (Scalable Vector Graphics) 闪亮登场！

SVG 不是由像素拼成的图片，它是用一堆数学代码 (XML) 画出来的图形。

  • 无限缩放不失真。
  • 支持多色彩。
  • 可以直接写在 HTML 中，用 CSS 或 JS 还能控制它做动画！

8.11.5 👨‍💻 任务 9：引入 Iconfont 字体图标

假设你在阿里图标库选好了“购物车”图标，如何在代码里使用？

<!-- 1. 引入 Iconfont 提供的在线 CSS 文件 -->
<link rel="stylesheet" href="//at.alicdn.com/t/font_xxx.css">

<!-- 2. 像使用普通 class 一样召唤图标 -->
<i class="iconfont icon-shopping-cart"></i>

/* 3. 在你自己的 CSS 中用文字属性控制它 */
.icon-shopping-cart {
    font-size: 50px; /* 放大 */
    color: red;      /* 瞬间变红！ */
}

8.11.6 💡 图像方案大比拼总结

我们在实际进行网页元素设计时，到底该选哪种方案？

  • 复杂的色彩照片或背景： 老老实实用 JPG / PNG / WebP 格式。
  • 大量的单色小图标（如侧边栏导航）： 强烈推荐 字体图标 (Iconfont)，轻量且好变色。
  • 高精度的多色Logo或图表： 首选 SVG 代码直接嵌入，最高清且支持交互！

忘掉雪碧图吧，现在是 SVG 和字体图标的时代！

8.12 display: table 老牌神器 📊

在 Flexbox 没普及的黑暗时代，想把一段文字在一个盒子里绝对垂直居中，比登天还难。

大家发现了一个偏方：HTML 里的 <table> 单元格自带强大的垂直居中属性 (vertical-align: middle)。

于是我们用 CSS 强行给普通 div 洗脑：display: table;。

让外层 div 假装自己是一个表格，内层 div 假装是单元格，从而蹭到了它的垂直居中能力！

👨‍💻 任务 10：用 Table 特性实现垂直居中

虽然现在都用 Flex 了，但在维护老项目时你会经常看到这个魔法。

<!-- HTML 结构 -->
<div class="parent">
    <div class="child">几行字
我都在正中间</div>
</div>

/* CSS 代码 */
.parent {
    display: table; /* 父元素扮作表格 */
    width: 300px; height: 300px;
}
.child {
    display: table-cell; /* 子元素扮作单元格 */
    vertical-align: middle; /* 经典的垂直居中！ */
}

8.13 终极排版杀器：Grid 网格布局 🍰

如果说用 float 布局像是在玩俄罗斯方块（到处拼凑）。

那么现在的 Grid (网格) 布局就像是在切生日蛋糕！

你可以直接告诉浏览器：“我要把这块区域切成 3 行 3 列，中间留 20px 缝隙。”

  • Flexbox (弹性盒) 是一维布局（擅长管好一行或一列里的元素）。
  • Grid (网格) 是真正的二维布局（能够同时掌控行和列的宏大版图）！

8.13 划分布局与 fr 单位 🎨

Grid 引入了一个极度好用的新长度单位：fr (fraction，份额)。

它代表容器剩余空间的“比例份额”。不用再头疼计算百分比了！

比如属性：grid-template-columns: 1fr 2fr 1fr;

意思是把宽度切成三份，第一列占1份宽，第二列占2份宽，第三列占1份宽。简单粗暴！

还有网格间距神器：gap: 20px;，一键解决元素之间的缝隙问题。

👨‍💻 任务 11：三行代码排版九宫格照片墙

将 9 张图片完美排列成 3x3 矩阵，且带有均匀的间距。

/* 只需要给外层的父容器设置样式即可！ */
.gallery {
    display: grid; /* 开启网格魔法 */
    
    /* 划分为 3 列，每列平均分配 (各占 1 份) */
    grid-template-columns: 1fr 1fr 1fr; 
    
    /* 照片之间的间距：20px */
    gap: 20px; 
}

里面的子元素 img 完全不需要写布局代码，自动乖乖入座，这就是 Grid 的降维打击！

8.14 CSS 变量：找个管家管颜色 📒

想象一下，你写了上千行 CSS，里面满屏都是你的品牌主题色 #1a73e8（蓝色）。有一天，老板说：“我觉得绿色更环保，全改成绿色吧！”你会崩溃的。

如果 CSS 能像编程语言一样声明一个变量就好了：定义 主题色 = 蓝色;

好消息是，CSS 原生支持自定义变量了！

  • 定义变量：名字必须以两个减号 -- 开头。
  • 使用变量：使用 var(--变量名) 调用。

👨‍💻 任务 12：一键切换主题色

在根节点统一定义颜色，全局调用，为“暗黑模式”打基础。

/* 1. 在根伪类 :root 中统一定义变量 (全局管家) */
:root {
    --main-color: #1a73e8; /* 蓝色 */
    --box-padding: 20px;
}

/* 2. 任何地方都可以使用这个变量 */
.button { background-color: var(--main-color); }
.title { color: var(--main-color); }
.box { padding: var(--box-padding); }

未来老板再提需求，你只需修改 :root 里的那一行代码即可下班！

8.15 网页性能优化：重排与重绘 🏗️🎨

浏览器把你写的代码渲染成画面，需要费很大力气。这其中最耗费性能的两个动作叫做：

  • 重排 (Reflow) = 拆墙重盖： 你改变了元素的宽高、位置（如 width, left），导致浏览器必须重新计算所有元素的几何位置，牵一发而动全身，极其消耗性能！卡顿的元凶！
  • 重绘 (Repaint) = 重新刷漆： 你只改变了颜色、背景（如 color）等不影响布局的属性。浏览器只需重新涂上新颜色，性能消耗较小。

👨‍💻 任务 13：写出丝滑不卡顿的动画

在做位移动画时，不要使用 left / top，这会引起疯狂的重排。

/* ❌ 错误做法：改变 left 引起重排，动画卡顿手机发热 */
.box:hover {
    left: 100px;
    width: 200px;
}

/* ✅ 高手做法：使用 transform，只引起重绘，还能开启GPU硬件加速 */
.box:hover {
    transform: translateX(100px) scaleX(2);
}

记住：动画首选 transform 和 opacity，这是前端高手的基本修养！

🏆 第8章 知识点大阅兵

今天我们一起解锁了许多高级魔法，大家还记得吗？

  1. 选择器不仅能选身份，还能选状态 (伪类) 和凭空造物 (伪元素)。
  2. 动起来要平滑 (transition)，自动播放靠导演 (animation)。
  3. 响应式移动优先靠媒体查询，自适应大小靠 rem 乘法。
  4. 忘掉雪碧图，拥抱 Iconfont 和 SVG 矢量图。
  5. 布局从远古 Table 进化到了终极 2D 杀器 Grid。
  6. 统一管理颜色用变量，改变位置用 transform 防止“重排”卡顿。
