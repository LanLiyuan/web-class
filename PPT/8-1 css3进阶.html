8.1 什么是 CSS3？

理论篇
“想象一下，如果你以前用 CSS2 做网页就像是在给毛坯房刷大白墙，那么有了 CSS3，你就是全副武装的高级室内设计师，直接自带全息投影和智能家居功能！”

CSS3 是层叠样式表的最新标准，它不仅向后兼容，还带来了爆炸级别的新特性：

  • 更平滑的圆角、阴影、渐变
  • 强大的2D/3D 转换与动画，取代繁琐的 JS 动画
  • 专为现代设备而生的媒体查询与弹性布局

8.2 移动网站设计：为什么要重视移动端？

理论篇

同学们，现在谁出门还不带手机？所以，我们的网页必须在手机上看得很舒服！

  • 自适应布局：服务器根据客户端设备返回不同的页面结构。
  • 响应式布局：开发一套代码，通过媒体查询自动适应从 PC 到手机的所有屏幕。
  • 核心设计原则：流式网格（Fluid Grid）、响应式图片、对触摸操作友好。
Page 4

8.2 移动网站的“敲门砖”：Meta Viewport

核心配置
“如果不加这行代码，手机浏览器会把 PC 网页硬生生缩小塞进屏幕，字小得要拿放大镜看！”

在 HTML 的 <head> 标签中，这行代码是响应式网站的灵魂：

🔧 核心代码
<meta name="viewport" content="width=device-width, initial-scale=1.0">

解释：让视窗宽度等于设备宽度，初始缩放比例为 1.0。

8.3 Flex 弹性布局 (高能预警⚡)

绝对重点
“还记得被 float 和清除浮动支配的恐惧吗？忘掉它们！现在是 Flexbox 的时代！”

传统的 float 布局本质是用来做文字环绕图片的，用来做页面排版其实是“歪打正着”。

Flexbox (弹性盒子) 是专门为现代 UI 布局而生的。只需一个属性，子元素就像被装进了弹簧盒子，可以自由伸缩、排列和对齐。

在企业级开发中，无论是 PC 端还是移动端，Flex 已成为首选排版方式。

Page 6

8.3.1 最核心概念：主轴与侧轴

理论篇

搞懂了这两个轴，Flex 布局你就掌握了一半！

“大家吃过烤面筋吗？主轴 (Main Axis) 就是穿面筋的那根竹签，元素就是面筋，顺着竹签串。侧轴 (Cross Axis) 就是垂直于竹签的方向！”
  • 主轴： 弹性盒子元素默认沿着主轴排列。
  • 侧轴： 永远垂直于主轴。
  • 注意： 主轴不一定总是水平的！我们可以随时把竹签竖过来。
Page 7

8.3.2 召唤弹性魔法：display: flex

容器属性

只要给父元素（容器）设置 display: flex，它里面的子元素，以前的“块元素独占一行”规则全部失效，全部乖乖遵守 Flex 规则！

👨‍💻 任务 1：创建你的第一个弹性容器
.container {
    display: flex; /* 就这一句，魔法开始生效 */
    background-color: #eee;
    height: 150px;
}

8.3.2 改变主轴方向：flex-direction

容器属性
“这根串面筋的竹签，我们既可以横着拿，也可以竖着拿。这就是 flex-direction 的作用。”

可选值有四个：

  • row：默认值，主轴为水平方向，起点在左端。
  • row-reverse：主轴为水平方向，起点在右端（反向）。
  • column：主轴为垂直方向，起点在上沿。
  • column-reverse：主轴为垂直方向，起点在下沿（反向）。

🔧 核心代码
.container {
    display: flex;
    /* 设置主轴为垂直方向 */
    flex-direction: column; 
    height: 400px;
}

8.3.2 挤不下怎么办？flex-wrap

容器属性
“想象一张三人长椅，来了五个人。默认的 Flex 会让大家拼命缩着身子挤在一起。如果不希望他们挤，想让他们坐在后排的椅子上，就要用到 flex-wrap。”

默认情况下，Flex 容器里的所有元素都会排在一条线上（所谓“不换行”）。

  • nowrap（默认）：不换行，子元素宽度会被强行压缩。
  • wrap：自动换行，第一行在上方。
  • wrap-reverse：换行，第一行在下方。
Page 13

👨‍💻 任务 3：让挤爆的容器自动换行

代码实操

当容器宽度只有 300px，但里面有 5 个 100px 宽的盒子时：

🔧 核心代码
.container {
    display: flex;
    width: 300px; 
    /* 允许子元素在空间不足时换行 */
    flex-wrap: wrap; 
}
.item { width: 100px; height: 100px; }

8.3.2 主轴对齐魔法：justify-content

容器属性

子元素水平排列后，默认是依次靠左挤在一起的。如果我们想让它们居中、靠右、或者均匀分布在父盒子里，这就是 justify-content 的工作范围。

  • flex-start（默认）：左对齐。
  • flex-end：右对齐。
  • center：居中对齐。
  • space-between：两端对齐，中间元素间距相等。
  • space-around：每个子元素两侧间距相等。

👨‍💻 任务 4：实现“两端对齐”导航栏结构

代码实操

在企业级开发中，最常用的就是 space-between，比如常见的“左侧 Logo，右侧菜单”布局。

🔧 核心代码
.navbar {
    display: flex;
    /* 主轴方向（水平）两端对齐，中间留白 */
    justify-content: space-between; 
    background-color: #333;
    padding: 0 20px;
}

8.3.2 侧轴对齐魔法：align-items

容器属性
“主轴对齐搞定了，侧轴（默认是垂直方向）呢？曾经让前端工程师头疼不已的‘垂直居中’，在 Flex 面前就是一句话的事！”

align-items 决定了元素在侧轴上的对齐方式（针对单行）：

  • flex-start：上对齐（起点对齐）。
  • flex-end：下对齐（终点对齐）。
  • center：垂直居中（这可是神器！）。
  • stretch（默认）：如果子元素未设置高度，将拉伸占满整个容器高度。
Page 19

👨‍💻 任务 5：实现完美的“绝对居中”

代码实操

终极任务：如何让一个盒子在一个大屏幕里，无论宽高，始终水平+垂直双向居中？

🔧 核心代码
body {
    display: flex;
    height: 100vh; /* 满屏高度 */
    justify-content: center; /* 水平（主轴）居中 */
    align-items: center;     /* 垂直（侧轴）居中 */
}

这三行代码，价值千金，请死死记住！

8.3.2 多行侧轴对齐：align-content

容器属性

align-items 是一行元素自己在自己的范围内垂直对齐。但是如果发生了换行（flex-wrap: wrap），有了好几行，这几行作为一个整体怎么在侧轴上对齐呢？这就需要 align-content。

“注意！align-content 对单行元素无效！它操作的是那些发生了换行产生的‘多行大团伙’。”

它的取值与 justify-content 类似：flex-start, flex-end, center, space-between, space-around。

Page 22

8.3.3 瓜分空间的艺术：项目属性 flex

子元素(项目)属性
“前面的属性都是加在父盒子身上的，现在这个属性 flex，是要写在里面的子盒子（项目）身上的！”

如果父盒子有剩余的空间，子盒子可以按照一定比例去瓜分它。使用 flex 属性时，宽度的设置（width）可能会失效，转而受弹性比例控制。

  • 语法：flex: 1;
  • 本质：它是 flex-grow, flex-shrink 和 flex-basis 的简写。
Page 23

👨‍💻 任务 6：按比例瓜分剩余空间

代码实操

假如容器宽度是 1000px，我们放了三个盒子，我们要让它们分别占据 1/4、1/4 和 2/4 的空间：

🔧 核心代码
.box1 { flex: 1; background: pink; }
.box2 { flex: 1; background: skyblue; }
.box3 { flex: 2; background: yellow; }

这里的总份数是 1+1+2 = 4份。box3 直接拿走了总宽度的一半！

8.3.3 flex 属性的终极秘密

进阶理解

有时你会看到 flex 后面跟了三个值，例如：

.aside { flex: 0 0 200px; }

这三个值分别代表什么？

  1. 放大比例 (flex-grow)：默认为 0，即使有剩余空间也不放大。
  2. 缩小比例 (flex-shrink)：默认为 1，空间不足时按比例缩小。这里写 0 表示“坚决不缩小”。
  3. 基础尺寸 (flex-basis)：在分配多余空间之前，项目占据的主轴空间（死死咬住 200px）。

8.4 使用 Flex 实现自适应宽度

实战篇
“老板要求：网页左右两侧边栏宽度固定，中间的内容区域能随着屏幕的拉宽自动变大变宽。如果用以前的技术，你会写到怀疑人生。现在，看 Flex 如何秒杀这个需求！”

这就引出了大名鼎鼎的 1-3-1 经典自适应布局 (又称：圣杯布局/双飞翼布局的现代解法)。

  • 头部 (Header) 和 尾部 (Footer) 占满 100% 宽度。
  • 中间分为左中右三块。
  • 左右两块写死宽度。
  • 中间内容区设置 flex: 1，自动吃掉所有剩下的空间。

👨‍💻 任务 7：经典后台管理布局 (CSS篇)

代码实操

关键的魔法在于 .main 也是一个 Flex 容器，而中间的 section 设置了 flex: 1;。 

.main {
    display: flex; /* 让中间三块水平排列 */
    width: 100%;
}
.main aside {
    flex: 0 0 200px; /* 两侧定宽200px，不放大，不缩小 */
    background-color: #D3DCE6;
}
.main section {
    flex: 1; /* 中间自适应：霸占一切剩余空间！ */
    background-color: #E9EEF3;
}

8.4.2 自适应宽度中的“刺头”：图片

自适应宽度
“排版做好了，插进一张大图，‘砰’的一声，整个页面被大图撑爆破相了。这种情况眼熟吗？”

在自适应布局中，图片如果不加限制，会保持原生像素大小。要让图片跟着父盒子的宽度动态伸缩，必须要进行约束。

🔧 核心代码
img {
    max-width: 100%; /* 图片最大不超过父容器 */
    height: auto;    /* 保持原有纵横比，高度自适应 */
}
Page 31

8.4.3 带距离的自适应宽度布局

自适应宽度

平时我们做的列表，卡片与卡片之间是有间距的（Margin）。如果在 Flex 布局中，你想要完美的间距而不破坏整体宽度比例，该怎么做？

传统做法：给子元素加 margin-right: 10px;，然后给父元素加负边距 margin-left: -10px; 抵消最后一项的溢出。

现代魔法：使用 Flex 专属属性 gap！

Page 32

👨‍💻 任务 8：优雅地设置元素间距 (gap)

代码实操

只要在父容器上写一句 gap: 20px;，卡片之间自动保持距离，而卡片与容器边缘不会产生多余的边距！

🔧 核心代码
.card-list {
    display: flex;
    justify-content: space-between;
    gap: 20px; /* 卡片之间的间距为 20px */
}
.card { flex: 1; } /* 每张卡片宽度自适应平分 */

8.5 网页变魔术：CSS3 Transform 变形

动画与特效
“静态的网页太死板了！鼠标悬停时放大一点、转一圈，立刻就显得高端大气上档次！这就是 Transform 的魅力。”

transform 属性向元素应用 2D 或 3D 转换。该属性允许我们对元素进行旋转、缩放、移动或倾斜。

  • 这不仅仅是改变位置，它是通过计算机 GPU 加速运算的，性能极高，丝滑流畅。

8.5 基本变形 (2D 转换) 核心方法

动画与特效

牢记这三大基本操作，你就能组合出无数特效：

  • 平移：translate(Xpx, Ypx)，改变元素在 X/Y 轴的位置。
  • 缩放：scale(比例)，1 是原尺寸，1.2 是放大 1.2 倍，0.5 是缩小一半。
  • 旋转：rotate(角度deg)，例如 45deg 代表顺时针转 45 度。

👨‍💻 任务 9：让图片悬停时放大并微微旋转

代码实操

小米官网中，鼠标放上去商品卡片微微上浮，这就是用 Transform 实现的。我们来写一个稍微夸张点的效果。

🔧 核心代码
.box {
    transition: all 0.5s; /* 加上过渡动画，让变形丝滑 */
}
.box:hover {
    /* 组合变形：放大 1.2 倍，并顺时针旋转 10 度 */
    transform: scale(1.2) rotate(10deg);
}

8.5 突破次元壁：3D 变形

动画与特效
“2D 是在纸面上画画，3D 是直接给你一个立体魔方！我们在网页里增加了 Z 轴（指向你眼睛的方向）。”

要开启 3D 魔法，必须告诉父盒子“我要打开 3D 空间”：

  • 父元素开启 3D 视角：perspective: 800px; (表示你的眼睛距离屏幕 800 像素看过去，产生近大远小的透视效果)。
  • 关键 3D 方法：rotateX() (绕着横轴翻跟头), rotateY() (绕着竖轴转身)。

👨‍💻 任务 10：制作一面 3D 翻转卡片

代码实操
🔧 核心代码
.scene {
    perspective: 1000px; /* 在父级建立透视镜头 */
}
.card {
    transition: transform 1s;
    /* 保持内部的 3D 空间 */
    transform-style: preserve-3d; 
}
.scene:hover .card {
    /* 沿 Y 轴（竖轴）翻转 180 度 */
    transform: rotateY(180deg); 
}
